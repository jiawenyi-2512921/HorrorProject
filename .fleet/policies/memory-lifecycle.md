# Memory Lifecycle

## Memory layers

### Constitution

Stable project law. Read often. Rarely changes.

### Domain

Role-specific standards and known-good patterns.

### Mission

The current objective, acceptance criteria, and risk posture.

### Working set

Minimal relevant files and evidence for the active packet.

### Ephemeral scratch

Temporary reasoning only. Never treated as durable truth.

### Evidence

Verified facts promoted from approved output.

## Promotion rules

- Promote only verified facts
- Link promoted facts back to source artifacts
- Replace old summaries instead of appending contradictory versions

## Cleanup rules

- Destroy ephemeral scratch after completion
- Invalidate working sets on file changes
- Compress long traces into short summaries with references
- Remove stale summaries not touched in two cycles
- Never keep speculative notes as long-term truth

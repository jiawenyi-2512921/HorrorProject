# Evidence Placement Guide

## Overview

Evidence items provide narrative depth, reward exploration, and enhance player immersion. Strategic placement is crucial for pacing and storytelling.

## Evidence System

### Evidence Categories

1. **Required Evidence** (8 items)
   - Tied to main objectives
   - Progression-critical
   - Clear placement
   - High visibility

2. **Optional Evidence** (12-20 items)
   - Lore expansion
   - Exploration rewards
   - Hidden placement
   - Medium-low visibility

3. **Secret Evidence** (3-5 items)
   - Easter eggs
   - Achievement unlocks
   - Very hidden
   - Low visibility

### Evidence Types

#### Documents (40%)
- Notes, letters, reports
- Readable text
- Lore-heavy
- Easy to place

#### Audio Logs (25%)
- Voice recordings
- Character backstory
- Atmospheric
- Requires audio design

#### Physical Objects (20%)
- Items, tools, personal effects
- Visual storytelling
- Contextual meaning
- 3D asset required

#### Environmental Evidence (15%)
- Blood stains, scratches, symbols
- Integrated into level
- Subtle storytelling
- Art pass integration

## Placement Principles

### 1. Narrative Flow

Evidence should tell a coherent story when collected in order:

**Act 1: Setup** (Evidence 1-8)
- Introduce setting and characters
- Establish normalcy
- Hint at danger
- Build curiosity

**Act 2: Escalation** (Evidence 9-16)
- Reveal threat
- Show consequences
- Character development
- Rising tension

**Act 3: Revelation** (Evidence 17-23)
- Truth revealed
- Climax information
- Character fates
- Resolution hints

### 2. Spatial Distribution

#### Zone Coverage
- Every major zone: 2-4 evidence items
- Hidden areas: 1-2 bonus items
- Safe zones: 1 guaranteed item
- Danger zones: High-value items

#### Density Map (SM13 - 1300m²)

| Zone | Required | Optional | Secret | Total |
|------|----------|----------|--------|-------|
| Entry Hall | 1 | 2 | 0 | 3 |
| Main Corridor | 1 | 1 | 0 | 2 |
| West Wing | 2 | 4 | 1 | 7 |
| East Wing | 2 | 3 | 1 | 6 |
| Basement | 1 | 2 | 1 | 4 |
| Upper Floor | 1 | 3 | 1 | 5 |
| Exit Sequence | 0 | 1 | 0 | 1 |
| **Total** | **8** | **16** | **4** | **28** |

### 3. Visibility Tiers

#### Tier 1: Obvious (Required Evidence)
- Main path placement
- Good lighting (100+ lux)
- Clear sightlines
- Environmental cues
- **Discovery Rate**: 95%+

#### Tier 2: Discoverable (Optional Evidence)
- Off main path
- Moderate lighting (50-100 lux)
- Requires exploration
- Subtle cues
- **Discovery Rate**: 60-80%

#### Tier 3: Hidden (Secret Evidence)
- Secret areas
- Dark (< 50 lux)
- No obvious cues
- Thorough search required
- **Discovery Rate**: 20-40%

### 4. Risk vs. Reward

Higher risk areas should contain more valuable evidence:

**Low Risk** (Safe zones)
- Common evidence
- Basic lore
- Easy access
- Low narrative value

**Medium Risk** (Mixed zones)
- Important evidence
- Character development
- Moderate challenge
- Medium narrative value

**High Risk** (Danger zones)
- Critical evidence
- Major revelations
- Significant challenge
- High narrative value

## Placement Workflow

### Phase 1: Narrative Design
1. Write complete evidence story
2. Divide into 28 pieces
3. Assign types to each piece
4. Create content for each item
5. Review narrative flow

### Phase 2: Spatial Planning
1. Map evidence to zones
2. Assign visibility tiers
3. Plan discovery order
4. Balance density
5. Mark on level map

### Phase 3: Implementation
1. Place evidence meshes/triggers
2. Implement interactions
3. Add UI feedback
4. Test discoverability
5. Adjust based on playtesting

### Phase 4: Polish
1. Add environmental cues
2. Refine lighting
3. Audio implementation
4. Final playtesting
5. Documentation

## Environmental Cues

### Visual Cues
- **Lighting**: Subtle highlights (rim lighting)
- **Color**: Distinct from environment (paper white, metal glint)
- **Composition**: Frame in doorways or sightlines
- **Props**: Contextual placement (desk, shelf, floor)

### Audio Cues
- **Proximity**: Subtle sound when near (paper rustle, static)
- **Interaction**: Clear pickup sound
- **Ambient**: Environmental context
- **Music**: Subtle musical sting on discovery

### Spatial Cues
- **Landmarks**: Near memorable locations
- **Paths**: Along natural exploration routes
- **Contrast**: Stand out from environment
- **Accessibility**: Reachable without frustration

## Interaction Design

### Pickup Interaction
```
1. Player approaches (< 2m)
2. Highlight appears (outline/glow)
3. UI prompt shows ("Press E to examine")
4. Player presses E
5. Evidence UI opens
6. Content displays (text/audio/3D model)
7. Player reads/listens
8. Player closes UI
9. Evidence marked as collected
10. Objective/achievement updates
```

### Evidence UI

#### Document Display
- Full-screen overlay
- Readable text (16pt minimum)
- Zoom capability
- Background blur
- Close button (ESC or click)

#### Audio Log Display
- Compact UI (bottom corner)
- Waveform visualization
- Play/pause controls
- Transcript option
- Continues during gameplay

#### Physical Object Display
- 3D viewer (rotatable)
- Zoom controls
- Inspection mode
- Annotations/hotspots
- Close button

### Collection Tracking

#### UI Elements
- Evidence counter (X/28)
- Category breakdown (Documents: X/11, Audio: X/7, etc.)
- Zone completion (West Wing: X/7)
- Secrets found (X/4)

#### Rewards
- **25% collected**: Achievement unlock
- **50% collected**: Bonus content unlock
- **75% collected**: Alternate ending hint
- **100% collected**: True ending unlock

## Performance Considerations

### Per-Evidence Budget
- **Draw Calls**: +10-20
- **Triangles**: +500-2000
- **Texture Memory**: +1-5MB
- **Audio Memory**: +500KB-2MB (per audio log)

### Optimization
1. Use instanced meshes for common evidence types
2. LODs for physical objects
3. Texture compression for documents
4. Audio streaming for logs
5. Disable highlights when far from player

## Common Mistakes

### 1. Too Hidden
- **Problem**: Players miss required evidence
- **Solution**: Increase visibility, add cues
- **Test**: Blind playtesting

### 2. Too Obvious
- **Problem**: No exploration challenge
- **Solution**: Reduce cues, move off main path
- **Test**: Experienced player feedback

### 3. Narrative Gaps
- **Problem**: Story doesn't make sense
- **Solution**: Review evidence order
- **Test**: Read all evidence in sequence

### 4. Clustering
- **Problem**: All evidence in one area
- **Solution**: Distribute across zones
- **Tool**: `ValidateLevelLayout.ps1`

### 5. Poor Readability
- **Problem**: Text too small or unclear
- **Solution**: Increase font size, improve contrast
- **Test**: Accessibility review

### 6. Audio Issues
- **Problem**: Audio logs hard to hear
- **Solution**: Balance levels, add subtitles
- **Test**: Audio mixing pass

## Narrative Coherence Check

### Story Arc Validation
1. Read all evidence in order
2. Verify story makes sense
3. Check for plot holes
4. Ensure character consistency
5. Validate timeline

### Pacing Check
1. Early evidence: Setup (engaging but not overwhelming)
2. Mid evidence: Development (reveals and escalation)
3. Late evidence: Payoff (satisfying conclusions)

### Optional Content Check
1. Optional evidence enhances but not required
2. Secrets provide bonus depth
3. No critical info in optional evidence
4. All paths narratively satisfying

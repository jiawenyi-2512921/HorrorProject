# 日志收集指南

## 如何收集日志文件

### 1. 游戏日志
**位置**: `[ProjectRoot]/Saved/Logs/HorrorProject.log`

**收集方法**:
1. 复现Bug
2. 立即关闭游戏
3. 找到上述路径的日志文件
4. 复制整个文件或相关片段

**关键信息**:
- 错误信息（Error）
- 警告信息（Warning）
- 断言失败（Assertion failed）
- 异常信息（Exception）

### 2. 崩溃报告
**位置**: `[ProjectRoot]/Saved/Crashes/`

**收集方法**:
1. 游戏崩溃后，会自动生成崩溃报告
2. 进入Crashes文件夹
3. 找到最新的崩溃文件夹（按时间排序）
4. 复制整个文件夹或其中的 `.log` 和 `.dmp` 文件

**重要文件**:
- `CrashContext.runtime-xml`: 崩溃上下文
- `CrashReportClient.ini`: 崩溃报告配置
- `*.log`: 崩溃日志
- `*.dmp`: 内存转储文件（用于深度分析）

### 3. 编辑器日志
**位置**: `[ProjectRoot]/Saved/Logs/`

**文件名**:
- `HorrorProject.log`: 最新日志
- `HorrorProject-backup-[timestamp].log`: 历史日志

### 4. 性能分析数据
**位置**: `[ProjectRoot]/Saved/Profiling/`

**收集方法**:
1. 在游戏中按 `~` 打开控制台
2. 输入 `stat startfile`
3. 复现Bug
4. 输入 `stat stopfile`
5. 在Profiling文件夹中找到生成的 `.ue4stats` 文件

### 5. 截图和视频
**截图位置**: `[ProjectRoot]/Saved/Screenshots/`

**录制方法**:
- 使用UE内置录制: 控制台输入 `StartMovieCapture` 和 `StopMovieCapture`
- 使用第三方工具: OBS、GeForce Experience、Xbox Game Bar等

## 日志分析技巧

### 查找错误信息
在日志文件中搜索以下关键词：
- `Error:`
- `Warning:`
- `Assertion failed`
- `Exception`
- `Fatal`
- `Crash`

### 查找性能问题
搜索以下关键词：
- `Hitch`
- `Long frame time`
- `Memory`
- `GC` (Garbage Collection)

### 查找资产问题
搜索以下关键词：
- `Failed to load`
- `Missing`
- `Invalid reference`
- `Unresolved`

# Bug报告最佳实践

## DO（应该做的）

1. **清晰的标题**: 用简洁的语言描述问题
   - 好: "角色在跳跃时穿过地板"
   - 差: "有个Bug"

2. **详细的复现步骤**: 让开发人员能够轻松复现
   - 包含每一个操作步骤
   - 说明操作的顺序
   - 提供必要的前置条件

3. **附加证据**: 截图、视频、日志
   - 一张图胜过千言万语
   - 视频能展示动态问题
   - 日志提供技术细节

4. **准确的环境信息**: 硬件、软件、设置
   - 有些Bug只在特定配置下出现
   - 帮助开发人员缩小范围

5. **及时更新**: 如果有新信息，及时补充
   - 发现新的复现方法
   - 找到临时解决方案
   - 验证修复结果

## DON'T（不应该做的）

1. **不要模糊描述**: "游戏有问题"、"不太对"
2. **不要省略步骤**: "然后就出Bug了"
3. **不要混合多个Bug**: 一个报告只描述一个问题
4. **不要情绪化**: 保持客观、专业
5. **不要重复报告**: 先搜索是否已有相同Bug

# Bug严重程度定义

## P0 - 致命
- 游戏无法启动
- 游戏频繁崩溃
- 数据丢失或损坏
- 安全漏洞
- 完全阻塞核心功能

**示例**:
- 启动游戏后立即崩溃
- 存档文件被破坏
- 主角无法移动

## P1 - 严重
- 核心功能无法使用
- 严重影响游戏体验
- 大范围影响玩家
- 无合理的替代方案

**示例**:
- 无法保存游戏
- 敌人AI完全失效
- 关键剧情无法触发

## P2 - 一般
- 功能异常但有替代方案
- 影响部分玩家
- 不影响核心体验
- 视觉或音频问题

**示例**:
- 某个UI按钮不响应（可用键盘替代）
- 特定场景下的贴图错误
- 非关键音效缺失

## P3 - 轻微
- 小的视觉问题
- 文本错误
- 不影响游戏性
- 优化建议

**示例**:
- 拼写错误
- 轻微的模型穿插
- UI对齐问题

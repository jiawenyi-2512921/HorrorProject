# Console Submission Guide
# 主机平台提交指南

## 概述

主机平台（PlayStation、Xbox、Nintendo Switch）提供了巨大的市场机会，但需要更严格的审核流程和技术要求。本指南涵盖主要主机平台的提交流程。

---

## 1. 通用准备

### 1.1 开发者资格

#### 基本要求
- [ ] 注册公司实体
- [ ] 商业计划和游戏概述
- [ ] 开发团队信息
- [ ] 财务能力证明
- [ ] 以往作品（如有）
- [ ] 专业网站和联系方式

#### 申请流程
1. 访问各平台开发者门户
2. 提交开发者申请
3. 等待审核（2-4周）
4. 签署开发者协议
5. 获得开发工具包访问权限

### 1.2 开发工具包

#### 硬件开发套件
- **PlayStation 5 DevKit** - 需要申请
- **Xbox Series X|S DevKit** - 需要申请
- **Nintendo Switch DevKit** - 需要申请

#### 软件开发套件
- 平台特定SDK
- 开发工具和文档
- 测试工具
- 提交工具

## 2. PlayStation平台

### 2.1 PlayStation Partner申请

#### 注册流程
- [ ] 访问 https://partners.playstation.net/
- [ ] 创建账户
- [ ] 提交公司信息
- [ ] 提交游戏概念
- [ ] 等待审核
- [ ] 签署协议

#### 所需信息
- 公司法律文件
- 游戏设计文档
- 开发时间表
- 预算计划
- 营销计划
- 团队资历

### 2.2 技术要求

#### PlayStation 5技术规格
```
CPU: AMD Zen 2, 8核 @ 3.5GHz
GPU: AMD RDNA 2, 10.28 TFLOPS
RAM: 16GB GDDR6
Storage: 825GB SSD
```

#### 必需功能
- [ ] PlayStation Network集成
- [ ] 奖杯系统
- [ ] 活动卡片（Activities）
- [ ] 游戏帮助系统
- [ ] DualSense功能（触觉反馈、自适应扳机）
- [ ] 3D音频支持
- [ ] 快速恢复
- [ ] 活动追踪

#### 性能要求
- [ ] 稳定帧率（30fps或60fps）
- [ ] 快速加载时间
- [ ] 无严重bug
- [ ] 通过TRC（技术要求检查清单）

### 2.3 提交流程

#### 准备阶段
- [ ] 完成游戏开发
- [ ] 集成所有PSN功能
- [ ] 通过内部QA
- [ ] 准备提交材料

#### 提交材料
- [ ] 游戏构建（PKG格式）
- [ ] 年龄分级证书
- [ ] 商店资产（图像、视频）
- [ ] 游戏描述和元数据
- [ ] TRC自检报告
- [ ] 测试报告

#### 审核流程
1. 技术审核（TRC）
2. 内容审核
3. 年龄分级验证
4. 商店页面审核
5. 最终批准

**审核时间：** 2-4周

### 2.4 PlayStation Store设置

#### 商店资产
- [ ] 游戏图标（多种尺寸）
- [ ] 背景图
- [ ] 截图（至少5张，1920x1080）
- [ ] 预告片（1080p或4K）
- [ ] 游戏描述（多语言）
- [ ] 年龄分级标识

#### 定价
- [ ] 设置基础价格（USD）
- [ ] 区域定价
- [ ] 折扣计划
- [ ] PS Plus折扣

## 4. Nintendo Switch

### 4.1 Nintendo Developer Portal申请

#### 注册流程
- [ ] 访问 https://developer.nintendo.com/
- [ ] 创建账户
- [ ] 提交开发者申请
- [ ] 提供公司和游戏信息
- [ ] 等待审核（可能较长）
- [ ] 签署协议

#### 注意事项
- Nintendo审核较严格
- 需要展示游戏质量
- 可能需要演示版本
- 审核时间较长

### 4.2 技术要求

#### Nintendo Switch技术规格
```
掌机模式:
CPU: ARM Cortex-A57, 4核 @ 1.02GHz
GPU: NVIDIA Maxwell, 157 GFLOPS
RAM: 4GB

底座模式:
CPU: ARM Cortex-A57, 4核 @ 1.02GHz
GPU: NVIDIA Maxwell, 393 GFLOPS
RAM: 4GB
```

#### 必需功能
- [ ] 掌机和底座模式支持
- [ ] 触摸屏支持（可选但推荐）
- [ ] Joy-Con控制器支持
- [ ] Pro控制器支持
- [ ] 睡眠模式支持
- [ ] 截图和视频录制
- [ ] Nintendo账户集成
- [ ] 云存档（Nintendo Switch Online）

#### 性能要求
- [ ] 两种模式稳定运行
- [ ] 合理的加载时间
- [ ] 电池续航优化
- [ ] 通过Lotcheck

### 4.3 提交流程

#### 准备阶段
- [ ] 完成Switch优化
- [ ] 测试两种模式
- [ ] 准备ROM文件
- [ ] 完成本地化（至少英语）

#### 提交材料
- [ ] 游戏ROM
- [ ] 年龄分级证书
- [ ] 商店资产
- [ ] 游戏手册
- [ ] Lotcheck表格
- [ ] 测试报告

#### Lotcheck流程
1. 提交到Nintendo
2. Lotcheck测试（严格）
3. 问题报告
4. 修复和重新提交
5. 最终批准

**Lotcheck时间：** 3-6周（可能更长）

## 5. 年龄分级

### 5.1 ESRB（北美）

#### 申请流程
- [ ] 访问 https://www.esrb.org/
- [ ] 创建账户
- [ ] 提交游戏信息
- [ ] 上传内容视频
- [ ] 支付费用
- [ ] 等待分级

#### 分级类别
- E (Everyone)
- E10+ (Everyone 10+)
- T (Teen)
- M (Mature 17+)
- AO (Adults Only 18+)

**恐怖游戏通常：** T或M

### 5.2 PEGI（欧洲）

#### 申请流程
- [ ] 访问 https://pegi.info/
- [ ] 通过授权机构申请
- [ ] 填写问卷
- [ ] 提交内容
- [ ] 支付费用
- [ ] 获得分级

#### 分级类别
- PEGI 3
- PEGI 7
- PEGI 12
- PEGI 16
- PEGI 18

### 5.3 其他地区

- **CERO（日本）** - 日本计算机娱乐分级组织
- **USK（德国）** - 德国娱乐软件自律机构
- **ACB（澳大利亚）** - 澳大利亚分级委员会
- **GRAC（韩国）** - 游戏物管理委员会

## 7. 发布策略

### 7.1 同步发布 vs 分阶段发布

#### 同步发布
**优势：**
- 统一营销
- 最大化初期影响
- 简化沟通

**挑战：**
- 协调多个平台
- 同时处理多个审核
- 资源密集

#### 分阶段发布
**优势：**
- 分散风险
- 学习和改进
- 专注资源

**挑战：**
- 延长营销周期
- 可能的社区不满
- 复杂的沟通

### 7.2 定价策略

#### 跨平台定价
- 保持价格一致性
- 考虑平台受众
- 区域定价差异
- 促销协调

#### 主机特定考虑
- PlayStation Plus折扣
- Xbox Game Pass影响
- Nintendo eShop促销
- 实体版定价（如有）

## 10. 最佳实践

### 做 ✓
- 尽早申请开发者资格
- 充分测试平台特定功能
- 遵守所有技术要求
- 准备充足的审核时间
- 与平台代表保持沟通
- 优化每个平台的体验
- 准备多语言支持
- 计划长期支持

### 不要 ✗
- 低估审核时间
- 忽视平台特定功能
- 提交未完成的构建
- 忽视本地化
- 违反内容政策
- 忽视性能优化
- 放弃发布后支持
- 忽视社区反馈

## 11. 时间线示例

### 典型主机发布时间线

```
T-12个月: 申请开发者资格
T-10个月: 获得开发工具包
T-9个月: 开始平台特定开发
T-6个月: 申请年龄分级
T-3个月: 完成开发，开始测试
T-2个月: 提交认证
T-1个月: 通过认证，准备发布
T-0: 发布日
T+1个月: 首个补丁
```

## 12. 成本估算

### 开发成本
- 开发工具包：$0-$2,500（取决于平台和计划）
- 年龄分级：$800-$4,000（每个地区）
- 本地化：$5,000-$50,000+（取决于语言数量）
- QA测试：$10,000-$50,000+
- 认证重新提交：时间和资源成本

### 发布成本
- 营销材料制作
- 公关和营销活动
- 社区管理
- 客户支持

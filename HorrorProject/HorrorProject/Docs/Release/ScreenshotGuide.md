# Screenshot Guide
# 游戏截图指南

## 截图的重要性

高质量的截图是营销材料的核心。它们是玩家在商店页面、社交媒体和媒体报道中看到的第一印象。优秀的截图可以显著提高转化率和愿望单添加率。

---

## 技术规格

### Steam要求
- **分辨率:** 1920x1080 (最低), 3840x2160 (推荐)
- **格式:** PNG或JPG
- **文件大小:** <5MB每张
- **数量:** 最少5张，推荐10-15张
- **比例:** 16:9

### Epic Games Store要求
- **分辨率:** 1920x1080 (最低), 3840x2160 (推荐)
- **格式:** PNG或JPG
- **文件大小:** <10MB每张
- **数量:** 最少4张，推荐8-12张
- **比例:** 16:9

### 通用最佳实践
- **分辨率:** 4K (3840x2160) 用于所有主截图
- **格式:** PNG用于UI清晰度，JPG用于文件大小
- **色彩空间:** sRGB
- **压缩:** 最小压缩，保持质量
- **命名:** 描述性文件名（如：Screenshot_MainHall_01.png）

## 截图类型和数量

### 必需截图（5-7张）
1. **标志性场景** - 最具代表性的游戏画面
2. **游戏玩法** - 展示核心玩法机制
3. **环境展示** - 展示游戏世界的美
4. **角色/敌人** - 展示关键角色或威胁
5. **氛围场景** - 展示游戏的情绪和基调
6. **UI展示** - 展示界面和交互（可选）
7. **特色功能** - 展示独特卖点（可选）

### 推荐额外截图（5-8张）
8. **不同环境** - 展示游戏多样性
9. **解谜场景** - 展示智力挑战
10. **紧张时刻** - 展示刺激和危险
11. **细节特写** - 展示视觉质量
12. **光照展示** - 展示技术实力
13. **对比场景** - 展示游戏范围
14. **社区最爱** - 玩家喜欢的场景
15. **更新内容** - 新增内容（持续更新）

## 截图拍摄指南

### 准备工作

#### 游戏设置
- [ ] 设置为最高图形质量
- [ ] 分辨率：3840x2160或更高
- [ ] 抗锯齿：最高设置
- [ ] 纹理质量：史诗/超高
- [ ] 阴影质量：史诗/超高
- [ ] 后期处理：最高
- [ ] 特效质量：最高
- [ ] 视野（FOV）：90-100（根据场景调整）
- [ ] 运动模糊：关闭（截图用）
- [ ] 景深：根据需要调整
- [ ] 色差：关闭或最小

#### 工具准备
- [ ] 使用游戏内截图工具或
- [ ] 使用虚幻引擎的高分辨率截图命令
- [ ] 准备自由相机模式
- [ ] 准备UI隐藏命令
- [ ] 准备时间暂停功能
- [ ] 准备光照调整工具

#### 虚幻引擎截图命令
```
// 基础截图
HighResShot 3840x2160

// 超高分辨率截图（用于印刷）
HighResShot 7680x4320

// 隐藏UI
ShowFlag.HUD 0

// 显示UI
ShowFlag.HUD 1

// 暂停游戏
Pause

// 自由相机
ToggleDebugCamera

// 调整时间
Slomo 0.1  // 慢动作
Slomo 1.0  // 正常速度
```

### 构图原则

#### 三分法则
- 将重要元素放在画面的三分之一处
- 避免将主体放在正中心（除非有意为之）
- 利用交叉点放置焦点

#### 引导线
- 使用走廊、道路、光线引导视线
- 创造深度感
- 引导观众注意力到重要元素

#### 框架构图
- 使用门框、窗户、拱门作为自然框架
- 增加深度和聚焦
- 创造"窥视"的感觉（适合恐怖游戏）

#### 对称与不对称
- 对称：创造秩序感、不安感
- 不对称：创造动态感、紧张感
- 根据场景情绪选择

#### 前景、中景、背景
- 确保三个层次都有内容
- 创造深度和立体感
- 前景可以框架化或引导

---

### 光照技巧

#### 自然光照
- 利用游戏内的光源
- 保持真实感
- 展示游戏实际外观

#### 戏剧性光照
- 强烈的明暗对比
- 侧光或背光
- 创造情绪和氛围

#### 色彩温度
- 暖色调：舒适、危险、火焰
- 冷色调：不安、神秘、月光
- 对比色温：视觉冲击

#### 光线方向
- 顶光：戏剧性、不祥
- 侧光：纹理、深度
- 背光：轮廓、神秘
- 底光：不自然、恐怖

### 场景选择

#### 标志性场景
**目标:** 一眼就能识别的游戏画面

**特征:**
- 独特的视觉元素
- 强烈的氛围
- 代表游戏核心体验
- 令人难忘

**示例场景:**
- 主大厅的宏伟景观
- 标志性的走廊或房间
- 独特的建筑特征
- 关键的故事地点

**拍摄技巧:**
- 选择最佳角度展示空间
- 确保光照完美
- 包含标志性元素
- 多拍几张不同角度

---

#### 游戏玩法场景
**目标:** 展示玩家实际在做什么

**特征:**
- 展示核心机制
- 包含UI（适度）
- 展示互动
- 清晰易懂

**示例场景:**
- 检查物品
- 解谜过程
- 阅读笔记
- 操作机关
- 使用工具

**拍摄技巧:**
- 选择教程或早期关卡（避免剧透）
- UI清晰可见但不遮挡
- 展示玩家视角
- 捕捉动作瞬间

---

#### 环境展示
**目标:** 展示游戏世界的美和细节

**特征:**
- 视觉冲击力
- 展示技术实力
- 展示美术风格
- 创造氛围

**示例场景:**
- 大型开放空间
- 精细装饰的房间
- 独特的建筑
- 自然环境（如有）
- 天气效果

**拍摄技巧:**
- 使用广角展示空间
- 注意光照和阴影
- 包含细节和纹理
- 展示深度和规模

---

#### 角色/敌人展示
**目标:** 展示威胁和角色设计

**特征:**
- 清晰展示设计
- 创造紧张感
- 不完全剧透
- 令人印象深刻

**示例场景:**
- 远处的威胁（神秘）
- 追逐场景（紧张）
- 对峙时刻（戏剧性）
- 部分展示（恐怖）

**拍摄技巧:**
- 使用光影隐藏部分细节
- 捕捉威胁感
- 避免完全清晰的正面照（保持神秘）
- 展示规模和危险

---

#### 氛围场景
**目标:** 传达游戏的情绪和基调

**特征:**
- 强烈的情绪
- 视觉叙事
- 艺术性
- 令人难忘

**示例场景:**
- 孤独的走廊
- 诡异的光影
- 废弃的房间
- 不祥的细节

**拍摄技巧:**
- 专注于情绪而非信息
- 使用戏剧性光照
- 简化构图
- 创造不安感

## 后期处理

### 基础调整
- **亮度/曝光:** 确保细节可见但保持氛围
- **对比度:** 适度增强，不过度
- **饱和度:** 根据游戏风格调整
- **锐化:** 轻微锐化，增强细节
- **降噪:** 如果需要，轻微降噪

### 色彩分级
- **保持一致性:** 所有截图应有统一的色调
- **增强氛围:** 轻微调整色温和色调
- **避免过度:** 保持真实感
- **测试不同显示器:** 确保在各种屏幕上看起来都好

### 不要做的事
- ✗ 过度HDR效果
- ✗ 过度锐化（产生光晕）
- ✗ 过度饱和（不真实）
- ✗ 添加不在游戏中的元素
- ✗ 误导性的后期处理

### 推荐工具
- **Adobe Photoshop** - 专业级调整
- **Adobe Lightroom** - 批量处理
- **GIMP** - 免费替代品
- **Capture One** - 专业色彩管理

## 截图管理

### 文件命名规范
```
[游戏名]_[场景描述]_[编号]_[分辨率].png

示例：
HorrorProject_MainHall_01_4K.png
HorrorProject_Corridor_Chase_02_4K.png
HorrorProject_Puzzle_Library_01_4K.png
```

### 文件夹结构
```
Screenshots/
├── Master/              # 原始4K截图
│   ├── Gameplay/
│   ├── Environment/
│   ├── Characters/
│   └── UI/
├── Steam/               # Steam优化版本
├── Epic/                # Epic优化版本
├── Press/               # 新闻资料包版本
├── Social/              # 社交媒体版本
│   ├── Twitter/
│   ├── Instagram/
│   └── Facebook/
└── Archive/             # 未使用的截图
```

### 版本控制
- 保留原始未处理版本
- 为不同平台创建优化版本
- 记录后期处理设置
- 定期备份

## 平台特定优化

### Steam商店页面
**主截图（第一张）:**
- 最具冲击力的画面
- 清晰展示游戏类型
- 避免过多UI
- 在缩略图中也要好看

**后续截图:**
- 展示游戏多样性
- 混合游戏玩法和环境
- 按重要性排序
- 定期更新（添加新内容）

**技巧:**
- 前3张最重要（首屏可见）
- 避免相似的截图
- 包含一张UI/功能展示
- 考虑添加简短文字说明（可选）

### Epic Games Store
**要求类似Steam，但注意:**
- 更注重视觉冲击
- 可以更艺术化
- 确保在深色背景上好看
- 考虑添加简短标题

### 社交媒体

#### Twitter/X
- **尺寸:** 1200x675 (16:9)
- **格式:** JPG（更快加载）
- **内容:** 单一焦点，清晰信息
- **文字:** 可以添加简短文字覆盖

#### Instagram
- **Feed:** 1080x1080 (1:1)
- **Stories:** 1080x1920 (9:16)
- **格式:** JPG
- **内容:** 视觉冲击，适合移动端
- **文字:** 大而清晰

#### Facebook
- **尺寸:** 1200x630
- **格式:** JPG
- **内容:** 清晰信息，吸引点击
- **文字:** 可以添加描述性文字

## 最佳实践总结

### 做 ✓
- 使用最高图形设置
- 拍摄4K或更高分辨率
- 多拍，精选
- 展示实际游戏画面
- 保持一致的视觉风格
- 定期更新截图
- 测试不同平台和设备
- 收集玩家反馈

### 不要 ✗
- 使用误导性画面
- 过度后期处理
- 包含占位符或开发UI
- 展示bug或故障
- 剧透关键情节
- 使用过时的截图
- 忽视构图原则
- 忘记备份原始文件

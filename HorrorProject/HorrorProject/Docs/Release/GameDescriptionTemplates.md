# Game Description Templates
# 游戏描述模板

## 描述长度指南

不同平台和用途需要不同长度的游戏描述：

- **超短版 (25-50字):** 社交媒体、广告
- **短版 (50-100字):** 商店页面简介、新闻标题
- **标准版 (150-250字):** 商店页面主描述
- **详细版 (300-500字):** 新闻稿、深度介绍
- **完整版 (500-1000字):** 官方网站、新闻资料包

---

## 超短版描述 (25-50字)

### 模板1：核心体验
```
在[设定]中体验[类型]的恐怖。[独特卖点]，
每个选择都影响你的命运。你能逃脱吗？
```

### 示例：
```
在废弃的精神病院中体验第一人称心理恐怖。
先进的AI敌人会学习你的策略，每个选择都影响你的命运。
你能逃脱吗？
```

---

### 模板2：问题导向
```
当[情境]时会发生什么？在[游戏名]中，
[核心玩法]决定你的生存。[独特特色]。
```

### 示例：

### 模板1：叙事导向

```
[开场钩子 - 引人入胜的问题或陈述]

[游戏名]是一款[类型]游戏，将玩家带入[设定]。
你扮演[角色]，[角色背景]。[初始情境]，
你必须[主要目标]。

[核心玩法段落]
游戏的核心是[主要机制]。[玩法描述1]。
[玩法描述2]。[玩法描述3]。每个决定都会
影响故事的发展和最终结局。

[特色段落]
[游戏名]特色：
• [特色1及描述]
• [特色2及描述]
• [特色3及描述]
• [特色4及描述]

[技术段落]
采用虚幻引擎5开发，游戏展现了[技术亮点]。
[视觉描述]。[音频描述]。

[结尾钩子]
[问题或挑战]。[行动号召]。
```

### 示例：

```
有些地方最好被遗忘。有些秘密应该永远埋藏。

HorrorProject是一款第一人称心理恐怖游戏，
将玩家带入废弃的Blackwood精神病院。
你扮演调查记者Sarah Chen，追踪失踪人口案件。
当你深入这座被诅咒的建筑时，你发现这里的
恐怖远超想象。你必须揭开真相，同时保持理智。

游戏的核心是探索和生存。搜索环境寻找线索，
解开复杂的谜题，躲避或智胜追捕你的威胁。
资源稀缺，每个决定都可能是生死之别。
游戏拥有多个结局，取决于你的选择和行动。

HorrorProject特色：
• 动态叙事系统 - 你的选择真正影响故事
• 学习型AI - 敌人会记住并适应你的策略
• 沉浸式环境 - 每个房间都讲述一个故事
• 心理恐怖 - 不仅仅是跳吓，而是持久的恐惧

采用虚幻引擎5开发，游戏展现了次世代的
视觉保真度。Lumen全局光照创造逼真的光影，
Nanite技术呈现惊人的细节。3D音频系统
让每个声音都成为线索或警告。

真相在黑暗中等待。但你准备好面对它了吗？
立即添加到愿望单，准备进入噩梦。
```

## 平台特定描述

### Steam商店页面

#### 简短描述 (最多300字符)
```
[一句话钩子]。[游戏类型和设定]。
[核心玩法]。[独特卖点]。[技术亮点]。
```

#### 关于这款游戏 (推荐500-1000字)

**结构：**
1. 开场段落（吸引注意）
2. 故事和设定
3. 核心玩法机制
4. 主要特色（使用列表）
5. 技术和视觉
6. 系统需求提示
7. 行动号召

**格式技巧：**
- 使用粗体突出关键词
- 使用列表增加可读性
- 分段清晰
- 包含情感化语言
- 避免过度营销用语

## 描述写作最佳实践

### 做 ✓
- 开场强力，立即吸引注意
- 使用主动语态
- 具体而非笼统
- 展示独特价值
- 包含情感元素
- 针对目标受众
- 包含行动号召
- 校对和编辑

### 不要 ✗
- 使用陈词滥调
- 过度承诺
- 使用行话（除非目标受众理解）
- 剧透关键情节
- 贬低竞品
- 使用全大写或过多感叹号
- 忽视SEO关键词
- 忘记本地化

## 本地化注意事项

### 文化适应
- 避免文化特定的习语
- 考虑不同市场的敏感性
- 调整幽默和语气
- 本地化示例和参考

### 长度调整
- 某些语言翻译后会更长
- 为扩展留出空间
- 测试所有平台的字符限制
- 准备多个长度版本

### 质量保证
- 使用专业翻译
- 母语者审核
- 测试游戏内上下文
- 保持品牌一致性

---

## 测试和优化

### A/B测试
- 测试不同的开场钩子
- 测试特色顺序
- 测试行动号召
- 测试描述长度

### 指标追踪
- 转化率（愿望单/购买）
- 点击率
- 页面停留时间
- 跳出率

### 持续优化
- 根据数据调整
- 收集用户反馈
- 监控竞品
- 定期更新
